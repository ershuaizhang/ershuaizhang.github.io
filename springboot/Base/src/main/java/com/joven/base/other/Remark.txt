说明：

项目的启动类 BaseApplication.java

确定每个文件路径下文件的作用：
    common:
        这个文件路径下放置一些工具类，或者公共的一些文件
    controller:
        这个文件夹下用来写所有的控制器，对应前端界面访问后端的对接的类的集合，按照不同的功能再分别建立文件路径或者通过文件名进行识别
    mapper
        这个里面是用来对接数据库的，对数据完成增删改查
    entity
        这里时数据模型分别对应前端的返回和请求的结构体的承载
    service
        这里时服务的实现区，主要的业务逻辑再这个里面进行书写

类的调用路径方法：
    1、前端发起 调用 controller 下的类，所承载数据的主体使用model中的进行
    2、controller 中调用 service 中的进行实现
    3、service调用 mapper 查询数据库或者是缓存
